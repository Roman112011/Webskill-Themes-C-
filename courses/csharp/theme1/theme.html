Встановлення dotnet CLI

Що таке dotnet (CLI) та C#?

C#, CLI, Dotnet, Dotnet CLI.

C#:C# (читається як "сі шарп") - це мова програмування, створена компанією Microsoft. Вона використовується для розробки різноманітних програм, включаючи десктопні програми, веб-додатки, ігри та багато іншого. C# є потужною та гнучкою мовою, яка допоможе вам створити різноманітні програми.

Dotnet:.NET - це платформа розробки, також розроблена компанією Microsoft. Вона надає засоби та середовище для створення, виконання та управління програмами. .NET містити багато різних компонентів та бібліотек, які полегшують розробку програм на різних платформах.

CLI:CLI - це скорочення від "command-line interface", що означає інтерфейс командного рядку. Це спосіб взаємодії з комп'ютером за допомогою текстових команд. Замість використання графічного інтерфейсу (GUI), ви можете вводити команди в командному рядку для виконання різних завдань.

Dotnet CLI (інтерфейс командного рядку .NET):Dotnet CLI - це інструмент командного рядка, який надає засоби для розробки, збирання та виконання програм на платформі .NET. Ви можете використовувати Dotnet CLI для створення нових проєктів, компіляції коду, запуску програм та багато іншого. Це потужний інструмент, який допоможе вам ефективно працювати з проєктами на C# та інших мовах програмування, що підтримуються .NET.

Ці інструменти дозволять вам ефективно вивчати та розробляти програми на мові C# з використанням платформи .NET, використовуючи командний рядок для взаємодії з комп'ютером та здійснення різних розробницьких завдань.

Створення Dotnet(c#) проєкту

Як створити свій перший Dotnet проєкт.

Основна команда у Dotnet CLI це dotnet, всі інші команди викликаються з неї:

  1. Команда dotnet - основна Dotnet CLI команда.
  2. Аргумент -h скорочено від 'help' - аргумент шо повідомляє команду dotnet про те що ми хочемо побачити список доступних команд.

Після виклику команди dotnet з аргументом -h ми бачимо список команд, але нас цікавлять тільки 3 команди з цього списку, а саме - build, new та run:

  1. build - компілює наш код в програму яку можна запустити та/або перенести на інший комп'ютер.
  2. new - дозволяє створити нову програму (створює папку з шаблоном).
  3. run - запускає нашу програму у тій командні строці у якій ми цю команду написали.

Почнемо з команди new, для того, щоб дізнатись як нею користуватись давайте використаємо -h, з цього списку нас цікавить аргумент --list.

Після виклику dotnet new --list ми побачимо список доступних для нас шаблонів програм:

  1. Команда яку ми викликали.
  2. Стандартний шаблон консольної програми.
  3. Назва шаблону яку потрібно використати з командою new щоб створити таку програму.

З початку використаємо команду cd для того, щоб перейти у папку яку ми створили з використанням mkdir:

  1. Команда cd яка дозволяє рухатись між папками, приймає: назву папки у яку ви хочете перейти або .. для того шоб перейти одну папку вверх.
  2. Назва папки яку ми створили раніше.

Якщо адреса змінилась - значить у вас вийшло успішно змінити папку:

А зараз ми створимо новий консольний(console)(нарешті) Dotnet проєкт за допомогою new:

  1. Основна команда у Dotnet.
  2. Команда для створення нового проєкту.
  3. Назва шаблону який ми хочемо використати.

Якщо все пройшло успішно то у вашій папці повинен появитись шаблон, який +- схожий на той що на скрнішоті, саме важливе це - Program.cs це основний файл вашої програми у якому ви будете писати код:

  1. Шлях до нашого проєкту у консолі та провіднику.
  2. Основний файл проєкту - Program.cs.
  3. Повідомленя про успішне створення проєкту.

Написання коду, компіляція та запуск

Зараз ми розглянемо шаблонний C# код, скомпілюємо та запустимо його.

У папці проєкту який ви тільки що згенерували повинен бути файл який називається Program.cs.

Program.cs - це основний файл у вашому C# проєкті, шаблонний код може відрізнятись, а саме:

Такий:

Console.WriteLine("Hello World!");

Або такий:

using System;

namespace YourProjectName
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello, world!");
        }
    }
}

Перший варіант новіший (спрощений), з часом я би рекомендував розібратись з другим. Але зараз нас це не цікавить.

В обох шаблонах знаходиться функція Console.WriteLine(), з string аргументом у "".

Для початку у консолі пропишіть наступну команду: dotnet run (консоль повинна бути відкрита у папці проєкту).run - компілює та запускає нашу програму, у консолі ви повинні побачити текст який знаходиться у лапках функції Console.WriteLine().

Змініть текст у "" та повторіть команду dotnet run, перевірте чи ваш текст вивівся у консоль.
